# 技术栈说明

> 本文档列出 YOUR_PROJECT_NAME 使用的核心技术栈及版本要求。

## 总览

| 类别 | 技术 | 版本 | 用途 |
|:-----|:-----|:-----|:-----|
| **运行时** | Node.js | >= 20.x | 服务端运行环境 |
| **框架** | — | — | 主框架 |
| **数据库** | PostgreSQL | >= 15 | 主数据存储 |
| **缓存** | Redis | >= 7.x | 缓存与会话 |
| **ORM** | — | — | 数据库访问层 |
| **前端** | — | — | 用户界面 |
| **构建工具** | — | — | 打包与编译 |
| **部署** | Docker | >= 24 | 容器化部署 |
| **CI/CD** | GitHub Actions | — | 自动化流水线 |

## 选型原则

1. **稳定性优先** — 优先选择 LTS 版本和社区成熟方案
2. **团队熟悉度** — 优先选择团队有经验的技术
3. **生态完整性** — 优先选择文档完善、插件丰富的方案
4. **可维护性** — 避免引入过度复杂的依赖链

## 依赖管理

- 使用 lockfile（`package-lock.json` / `pnpm-lock.yaml`）锁定版本
- 定期执行依赖安全审计（`npm audit` / `pnpm audit`）
- 重大版本升级需创建独立分支评估兼容性
